# Background Color Update Documentation

## Overview
Updated the main dark background color from `#08090A` to `#374152` while preserving all gradient effects and star animations.

## Changes Made

### Color Update
- **File**: `next/tailwind.config.ts`
- **Change**: Updated `charcoal` color from `#08090A` to `#374152`
- **Impact**: This color is used throughout the application as the main dark background

### Preserved Features
✅ **Star Background** - The animated star field component remains unchanged
✅ **Shooting Stars** - The shooting star animations continue to work
✅ **Gradient Effects** - All gradient overlays and effects are preserved
✅ **Ambient Colors** - The ambient color overlays remain intact

## Color Usage

### Primary Dark Mode Color
- **Main Background**: `#374152` (charcoal)
- **Used in**:
  - Main layout background (`bg-charcoal` class)
  - CSS variable `--charcoal` (automatically generated by Tailwind)
  - Gradient transitions
  - Card backgrounds
  - Various UI components

### Related Dark Mode Colors
- **Elevated Surfaces**: `#1f2937` (gray-800) - Used for elevated cards and badges
- **Text Colors**:
  - Primary: `gray-100` (not pure white)
  - Secondary: `gray-200`
  - Muted: `gray-300`
- **Borders**:
  - Default: `gray-600`
  - Light: `gray-700`
  - Medium: `gray-500`

## Technical Details

### Tailwind Configuration
The `charcoal` color is defined in `tailwind.config.ts` and automatically generates:
- Tailwind utility classes: `bg-charcoal`, `text-charcoal`, `border-charcoal`, etc.
- CSS variable: `--charcoal` (via `addVariablesForColors` function)

### Components Using Background Color
- `next/app/[locale]/layout.tsx` - Main layout wrapper
- `next/components/gradient-container.tsx` - Uses `var(--charcoal)` for gradient transitions
- `next/components/dynamic-zone/launches.tsx` - Uses `var(--charcoal)` in background array
- Various other components using `bg-charcoal` class

## Design Philosophy
The new color `#374152` follows Arc Publishing's gray-700 baseline approach, avoiding pure black/white for better visual comfort and reduced eye strain in dark mode.

## Color Adjustments (Follow-up)

After updating the main background to `#374152`, several color adjustments were made to maintain proper contrast and visibility:

### Elevated Surfaces
- **Previous**: `#1f2937` (gray-800) / `neutral-900`
- **New**: `#4B5563` (gray-600) - Custom color `elevated`
- **Used in**: Cards, navbar backgrounds, elevated UI elements
- **Reason**: With the lighter main background, elevated surfaces needed to be lighter to maintain visual hierarchy

### Text Gradients
- **Previous**: `from-neutral-800` in heading gradients
- **New**: `from-neutral-500` 
- **Files Updated**:
  - `components/elements/heading.tsx`
  - `components/products/product-items.tsx`
  - `components/products/featured.tsx`
- **Reason**: Darker gradient start colors were too dark against the new background

### Borders
- **Previous**: `neutral-800`, `neutral-700`, `neutral-600`
- **New**: 
  - `borderLight`: `#6B7280` (gray-500)
  - `borderMedium`: `#9CA3AF` (gray-400)
- **Used in**: Card borders, dividers, input borders
- **Reason**: Lighter borders provide better visibility and contrast

### Buttons
- **Muted Variant**: Updated from `neutral-800` to `elevated` color
- **Hover States**: Adjusted to use lighter colors for better feedback

### Components Updated
- Navbar (desktop & mobile)
- Cards (blog, pricing, features)
- Buttons
- Form inputs
- Borders and dividers
- Elevated surfaces throughout the app

### Additional Fixes (Second Follow-up)

Additional elements that were too dark after the initial changes:

1. **Large Numbered Sections (01, 02, etc.)**
   - **Previous**: `text-neutral-900`
   - **Updated**: `text-neutral-600` → `text-white`
   - **File**: `components/dynamic-zone/how-it-works/card.tsx`
   - **Reason**: Changed to white for maximum visibility and consistency with button text styling

2. **Section Backgrounds**
   - **Previous**: `var(--zinc-900)` in launches component
   - **New**: `var(--charcoal)` (consistent with main background)
   - **File**: `components/dynamic-zone/launches.tsx`
   - **Reason**: Maintains consistency and proper contrast

3. **Card Backgrounds**
   - **Previous**: `bg-[rgba(40,40,40,0.30)]`
   - **New**: `bg-[rgba(75,85,99,0.40)]`
   - **Files**: 
     - `components/dynamic-zone/features/card.tsx`
     - `components/dynamic-zone/testimonials/testimonials-marquee.tsx`
   - **Reason**: Lighter, more visible cards that maintain the glassmorphism effect

4. **Feature Icon Gradients**
   - **Previous**: `from-neutral-800 to-neutral-950`
   - **New**: `from-neutral-600 to-neutral-800`
   - **File**: `components/dynamic-zone/features/feature-icon-container.tsx`
   - **Reason**: Lighter gradient maintains visual interest while being visible

5. **Mission Numbers**
   - **Previous**: `text-neutral-800` → `text-neutral-500`
   - **Updated**: `text-white` (final)
   - **File**: `components/dynamic-zone/launches.tsx`
   - **Reason**: Changed to white to match button text styling for maximum visibility and consistency

Removed black backgrounds from testimonial components:
